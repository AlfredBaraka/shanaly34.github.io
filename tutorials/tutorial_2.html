Python provides powerful tools for working with files. Let's explore the essentials of file handling.

Importance of File Handling

File handling is crucial for reading and writing data to files, which is essential for data storage, retrieval, and manipulation in various applications.

Disadvantages of File Handling

While file handling is useful, it has some limitations, such as:

  • Limited scalability for large datasets
  • No built-in support for advanced data structures
  • Potential security risks, such as file injection attacks

Precautions

When working with files, it's essential to:

  • Handle exceptions gracefully to avoid crashing your program
  • Close files properly after use to prevent memory leaks
  • Sanitize input to mitigate security risks

Python Code for File Handling

Python provides built-in functions to perform various operations on files, including reading, writing, and appending data.

1. Writing to a File


# Open the file in write mode
with open("my_file.txt", "w") as file:
    # Write three lines of text to the file
    file.write("Hello, world!\n")
    file.write("This is line 2.\n")
    file.write("The answer is 42.\n")

2. Reading from a File


# Open the file in read mode
with open("my_file.txt", "r") as file:
    # Read the contents of the file
    content = file.read()
    # Display the contents on the console
    print(content)

3. Appending to a File


# Open the file in append mode
with open("my_file.txt", "a") as file:
    # Append three additional lines of text to the file
    file.write("This is line 4.\n")
    file.write("Appending more text.\n")
    file.write("Last line.\n")

Note: Make sure you have the text file (my_file.txt) when running the code.
